Overview

For this lesson's team activity, we'll focus on templating in NodeJS.

Using the EJS templating engine is recommended, as both the class Team Activity solutions and Online Videos Course will be using it often. However, Pug and Handlebars are valid alternatives.

Instructions

It’s advised to complete the reading from the Online Videos Course this lesson before starting the acitvity. It will contain a lot of information and resources that are relevant to this team activity.

Once you are ready, complete the following core challenges with your team:

  1. Modify the ta02.ejs page to display a form with method POST and action "/ta02/addUser". Also create a section of the ejs file that will loop through an array of at least 3 names sent over via the render method.

  2. Using the “/addUser” POST path, render push the new username from the form submission into the existing user array, and then redirect to "/ta02/" to display the modified array.

  3. Create another form, but this time using a new "/removeUser" endpoint that removes a specified name from the array.

Stretch Challenges

After your team has finished the core requirements above, you’re encouraged to go above and beyond and experiment more with templating. Here are some recommended tasks:

  1. Display an error message if a username is not found for removal.

  2. Prevent duplicate insertions and display an error message.

Tips

  • A const array can be declared at the top of your ta02.js file that is sent over via the render method for router.get('/').

  • In EJS, your templating brackets for displaying variables sent through the render method are <%= variableName %>, while embedded javascript uses these brackets: <% %> You can read more about EJS here.

  • Lesson 66: “Using Express Router” in the Online Videos Course provides a good overview of how to create and use routes using ExpressJS.

  • Nodemon is a helpful tool that automatically restarts your local server whenever a change is saved.
