Overview

For the team activity this lesson, you will be downloading the class file organization template. You will also learn how to start local development using your localhost (i.e localhost:5000). While the team activity for this lesson has already been completed in the template, it is expected that you and your team thoughtfully look over and discuss the code.

Below are steps to use a class template for file organization that will make your life a lot easier:

    Download the zip file of the class template. The template can be found here: cse341-project

    A few steps to complete the installation:

  1. Assure that you are currently in the cse341-project folder when you type in the following commands

  2. In the terminal type npm install express --save. This will install all the necessary files needed to use express.

  3. In the terminal type npm install nodemon --save. While not necessary 'nodemon' allows for your localhost server run again every time your files are saved.

  4. In order to start up a localhost session either type in nodemon if you installed nodemon or npm start if nodemon was not installed

  5. Type localhost:5000 in your browser url (This is a local workspace for you to test your code)

Once you have dowloaded the template complete the following steps with your team:

  1. Navigate to the ta01.js file found in the routes folder

  2. Together with your team thoughtfully examine the code

  3. Assure that all members of the team understand what the code is doing. To do this, ensure that you all can answer the following questions confidently.

    • What is the res object?

    • What does res.write do?

    • What is the routes directory used for?

    • What is the views directory used for?

    • What is the public directory used for?

  5. If your team is unsure about any one of these questions, please post on the #help channel in Microsoft Teams.

Finally, you're going to save all of this code in a new, personal repository on github. If you would like to use a graphical user interface like GitLens you may. Here, you will learn how to use git commands to complete this.

  1. Having completed the steps in the the course setup at the beginning of this lesson's reading, you know how to create a new repository in github.

  2. Create another new repository. This repository you will build on with each team activity and many assignments throughout the semester.

  3. After having created it in github, on your computer in this folder with the downloaded code, you will start a new git project, add the remote of the new repo in github, then add, commit and push all of your code to it. Some helpful commands are as follows:

    • git init

    • git remote add

    • git remote -v

    • git add

    • git commit

    • git push
