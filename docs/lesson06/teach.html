Overview

For your team activity this week, you'll review your teammates e-commerce web applications.
In most computer science or software engineering careers, you'll be asked to review the work of your peers and give feedback. For this review, you'll want to consider what requirements are met for the Prove 06 assignment.

The final application should contain functionality for:

  • Store Product CRUD (Create, Read, Update Delete) operations using Mongoose Schemas or a similar external backend (e.g. Firebase)
    This means a user can:

    • Add a product to the store.

    • Edit products they have added to the store.

    • Remove products they have added to the store.

    The user cannot remove or edit product's submitted by another user.

  • Authentication methods are used to:

    • Sign users up for the website

    • Log existing users into the website

    • Send a confirmation email to a newly created account's email.

    • Prevent duplicate accounts (email is always unique)

    • Prevent access to administrative web endpoints of other users (CRUD operations for products).

    • Prevent rendering of administrative web endpoints in the navigation bar given the user is not signed in.

While the applications produced may not be totally unique in purpose, some of your teammates may choose to go above and beyond. Basically, these are features that were not covered in the preparation material. Such features could include:

  • Optimizing the CSS for both mobile and desktop browsers.

  • A unique and professional looking CSS layout.

  • Utilizing AJAX requests through JavaScript rather than <form> actions/methods.

  • Adding extra fields to database items, such as stock quantity, and associated features (orders decrement item stock, users cannot purchase more than the available stock).

  • User has the ability to decrement the quantity of a certain item in a cart.

  • User has the ability to decrement the quantity of a certain item in a cart.

Make sure to list feedback in a comprehensive and constructive manner.

  1. Positive features (if any) with reasonable explanations as to why they are good should be listed first.

  2. Negative features (if any) with reasonable explanations as to why they are bad should be listed second.

  3. Finally, each negative feature should contain a recommendation for improvement.
