Using git in teams - mastering branches and the git workflow

The following videos will be very helpful in learning how git is used in a team/collaborative environment. As you prepare to work on your projects with your teams, you will need to create branches for new features that you intend to implement; then use requests to merge these completed feature branches with the master branch. It will take a bit of practice to get good at this process, but this is the exact process that is used in the professional industry when any software development takes place. You do not need to watch all of these videos, but once you start learning, you should start trying to implement these principles (creating branches) in your workflow whether you're working alone or with a team.
