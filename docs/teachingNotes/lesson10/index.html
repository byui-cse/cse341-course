CRUD Operations in a REST API

Not much will change for using CRUD operations in an API. The only real difference is that responses contain JSON data instead of a rendered page. Most of the logic will be the same as earlier in the course.

JWT Authentication

Because REST APIs are "stateless," we don't store sessions in our API server. We need a way to do stateless authentication. This is where JSON web tokens (JWT) come in.

What is JWT?

  • Tokens are generated by the server.

  • Token is stored on the client and sent with every request.

  • Tokens can only be validated by the server. Changes to the token will be detected by the server, so tokens can't be faked.

  • Tokens contain JSON data and a signature. The JSON data can be decoded by the client, but only the server can verify the signature.
