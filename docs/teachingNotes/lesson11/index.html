Websockets

Websockets allow for bi-directional communication between the client and the server. The client can emit events to the server, and the server can emit events to one or more clients. This allows for real-time communication between clients without needing to reload the page.

Source

Socket.io

Socket.io is a package that makes it easy to work with websockets in an Express.js app.

  • Socket.io needs to be used on both the frontend and the backend

    • Backend: npm install socket.io

    • Frontend: npm install socket.io-client

  • Socket.io uses your existing Express.js server to handle requests

GraphQL

GraphQL is an alternative to RESTful services for an API. It provides a number of advantages compared to REST, including a rich query language that allows the client to specify what parts of the data the server should send in the response.

Overview

  • GraphQL uses a single endpoint for all requests: POST /graphql

  • Create a mongoose-like schema that defines types, queries, and mutations

    • Queries are for retrieving data, like a GET request in a RESTful API

    • Mutations are for any changes made to data, like POST, PUT, PATCH, and DELETE requests in a RESTful API

  • Use GraphQL's query language in the request body to make requests on the frontend
